Consumer Driven Contract Testing

PACT 101

WTF is dis?

Contract testing is a methodology for ensuring that two separate systems (such as two microservices) are compatible and are able to communicate with one other.
Consumer driven contract testing is a type of contract testing that ensures that a provider is compatible with the expectations that the consumer has of it.
More info here

PACT 101

Terminology

Service consumer
A component that initiates an HTTP request to another component.
Service provider
A server that responds to an HTTP request from another component. Should be thought as the deployable unit.
Mock Service Provider
Used by tests in the consumer project to mock out the actual service provider, meaning that integration-like tests can be run without requiring the actual service provider to be available.
